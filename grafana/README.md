# Grafana Monitoring

This directory contains Grafana assets for the Real-Time Financial Fraud Detection System.

## Dashboards

### Fraud Alerts Monitoring Dashboard

Dashboard export file:

```text
grafana/dashboards/fraud_alerts_dashboard.json
```

This is the final Grafana dashboard created for monitoring fraud alerts produced by the real-time fraud detection pipeline.

Pipeline monitored by this dashboard:

```text
Kafka → Spark Structured Streaming → Delta Lake Gold Layer → PostgreSQL → Grafana
```

## Purpose

The dashboard is designed to monitor flagged fraud alerts, not the full transaction population.

The PostgreSQL `fraud_metrics` table receives records from the Gold layer, which contains flagged transactions only. Therefore, dashboard metrics are focused on fraud alert monitoring and investigation.

## Data Sources

The dashboard uses the PostgreSQL Grafana data source and reads from the following tables:

| Table | Purpose |
|---------|---------|
| fraud_metrics | Fraud alert metrics and scored transactions |
| dq_checks | Data quality monitoring results |
| model_training_history | ML model version and performance metrics |

## Main Dashboard Sections

The dashboard includes:

- Fraud alert KPIs
- Fraud score and severity distributions
- Highest alert amounts
- Top alerted entities
- Average ML vs rule-based detection contribution
- Model performance metrics
- Data quality checks
- Fraud alert detail table

## Dashboard Queries

The dashboard panels are powered by PostgreSQL queries against the project monitoring tables. The main queries used in the dashboard are summarized below.

### Fraud Alert KPIs

These queries calculate the key fraud monitoring metrics displayed at the top of the dashboard.

```sql
-- Total Fraud Alerts
SELECT COUNT(*) AS total_fraud_alerts
FROM fraud_metrics
WHERE transaction_id NOT IN ('TXN_001','TXN_002','TXN_003','TXN_004','TXN_005');

-- Average Fraud Score
SELECT ROUND(AVG(fraud_score)::numeric, 3) AS avg_fraud_score
FROM fraud_metrics
WHERE transaction_id NOT IN ('TXN_001','TXN_002','TXN_003','TXN_004','TXN_005');

-- High Risk Alerts
SELECT COUNT(*) AS high_risk_alerts
FROM fraud_metrics
WHERE transaction_id NOT IN ('TXN_001','TXN_002','TXN_003','TXN_004','TXN_005')
  AND fraud_score >= 0.8;
```

---

### Data Quality Monitoring

The Data Quality panel retrieves the latest validation results generated by the Airflow monitoring workflow.

```sql
SELECT
    check_timestamp,
    check_name,
    status,
    metric_value,
    threshold,
    details
FROM dq_checks
ORDER BY check_timestamp DESC
LIMIT 10;
```

---

### Model Performance Metrics

The dashboard displays the latest recorded machine learning evaluation metrics.

```sql
SELECT
    model_version,
    roc_auc,
    pr_auc,
    precision,
    recall,
    f1_score
FROM model_training_history
ORDER BY training_date DESC;
```

---

### Fraud Alert Details

The fraud alert table displays the highest-risk transactions together with their rule-based score, ML score, and final fraud score.

```sql
SELECT
    timestamp,
    transaction_id,
    card_id AS entity_id,
    amount,
    ROUND(fraud_score * 100, 1) AS fraud_score_pct,
    ROUND(rule_based_score * 100, 1) AS rule_score_pct,
    ROUND(ml_score * 100, 1) AS ml_score_pct
FROM fraud_metrics
WHERE transaction_id NOT IN ('TXN_001','TXN_002','TXN_003','TXN_004','TXN_005')
ORDER BY fraud_score DESC
LIMIT 20;
```

These queries ensure that the Grafana dashboard visualizes only validated fraud alerts generated by the production pipeline while excluding the five initialization sample records.

## Importing the Dashboard

If Grafana is reset or the local Grafana database is recreated, the dashboard can be restored manually:

1. Open Grafana.
2. Navigate to **Dashboards**.
3. Select **Import**.
4. Upload:

```text
grafana/dashboards/fraud_alerts_dashboard.json
```

5. Select the PostgreSQL data source.
6. Import the dashboard.
